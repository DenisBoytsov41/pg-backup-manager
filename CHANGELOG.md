# Changelog

Все значимые изменения проекта будут фиксироваться в этом файле.

## [0.1.0] - 2026-03-10

### Added
- Создан GitHub-репозиторий проекта
- Подготовлено виртуальное окружение Python
- Создан базовый каркас директорий проекта
- Добавлены стартовые файлы проекта
- Добавлен `README.md`
- Добавлен `.gitignore`
- Добавлен `.editorconfig`
- Добавлен `pyproject.toml`
- Добавлен базовый entrypoint `__main__.py`
- Добавлены архитектурные документы:
  - `project-structure.md`
  - `application-flow.md`
  - `config-model.md`
  - `scheduler-integration.md`

- Реализованы domain models для профилей backup и настроек приложения
- Добавлены валидаторы профилей и расписаний
- Добавлен JSON config store для загрузки и сохранения профилей
- Добавлены application services для работы с профилями и настройками приложения

- Добавлен модуль `shared.paths` для работы с путями приложения и именами файлов backup
- Добавлен `LoggingService` для ведения основного и run-log журналов
- Добавлен `BackupRunner` для выполнения резервного копирования PostgreSQL-баз через `pg_dump` и `pg_dumpall`
- Реализована очистка устаревших backup/log/sql файлов по сроку хранения
- Выполнена проверка импортов и базовой работоспособности слоёв `shared` и `infrastructure`

- Добавлен CLI entrypoint в `__main__.py`
- Реализованы команды `run-profile` и `validate-profile`
- Добавлены коды возврата CLI для ошибок конфигурации, валидации и backup

- Добавлен новый GUI `ui.main_window` на основе слоёв `app`, `domain` и `infrastructure`
- CLI entrypoint обновлён: по умолчанию теперь запускается GUI
- Добавлена загрузка и сохранение профилей через новый интерфейс
- Добавлен тестовый запуск backup из GUI
- Добавлено сохранение пользовательских настроек приложения
- Исправлена работа `pg_dumpall --globals-only`: SQL-файл globals теперь сохраняется отдельно от служебного вывода
- Улучшена работа с путями приложения в dev-режиме
- Улучшена типизация работы с `Entry`/`TEntry` в GUI

- Добавлен модуль `infrastructure.scheduler` для работы с `schtasks`
- Реализованы операции создания, удаления, запуска и запроса задач Планировщика
- Добавлен `SchedulerService` в application-слой
- Добавлена нормализация дней недели для еженедельных задач
- Добавлен базовый парсинг статуса задач Планировщика

- Планировщик задач подключён к GUI
- Добавлены кнопки создания, проверки, запуска и удаления задачи
- Добавлен вывод статуса задачи в интерфейсе
- Добавлена генерация команды запуска задачи для dev/exe-режимов
- Добавлено поле пароля пользователя запуска, не сохраняемое в профиль

- Добавлены CLI-команды для Планировщика задач:
  - `scheduler-create`
  - `scheduler-query`
  - `scheduler-run`
  - `scheduler-delete`
- Добавлен вывод статуса задачи Планировщика в CLI
- Добавлен отдельный код возврата для случаев, когда задача не найдена при `scheduler-query`

- Добавлен пример профиля `resources/examples/example-profile.json`
- Добавлено краткое руководство пользователя `docs/user-guide/quick-start.md`
- Добавлена инструкция по сборке `docs/dev-guide/build.md`

- Улучшена обработка ошибок в GUI при ручном запуске backup
- Ошибки валидации и выполнения backup теперь показываются пользователю через диалоговые окна
- Уточнён сценарий тестового запуска backup при неверном пути к `pg_dump.exe`

## [0.1.1] - 2026-03-11

### Changed
- Усилена валидация профилей backup и настроек Планировщика задач
- Добавлена проверка шаблона `naming_pattern`
- Добавлена корректная санитизация имён файлов под Windows
- Реализовано фактическое использование `naming_pattern` при создании dump-файлов
- Уточнена логика работы `dump_globals` и проверки пути к `pg_dumpall.exe`
- Улучшена обработка путей профилей и recent profiles
- Уточнена документация по модели конфигурации, quick start и интеграции с Планировщиком задач
