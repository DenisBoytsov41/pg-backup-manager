# Application Flow

## Назначение документа

Этот документ описывает общий поток работы приложения PG Backup Manager.

## Основные пользовательские сценарии

### 1. Запуск приложения
Пользователь открывает GUI-приложение.
Приложение:
- определяет рабочие пути;
- загружает настройки приложения;
- открывает последний использованный профиль или пустую форму.

### 2. Создание профиля backup
Пользователь:
- создаёт новый профиль;
- заполняет параметры PostgreSQL;
- указывает директорию backup;
- указывает параметры retention;
- сохраняет профиль.

### 3. Тестовый запуск backup
Пользователь нажимает кнопку тестового запуска.
Приложение:
- валидирует профиль;
- вызывает application-сервис запуска backup;
- infrastructure-слой вызывает pg_dump / pg_dumpall;
- лог записывается в файл;
- результат отображается пользователю.

### 4. Настройка регламентного запуска
Пользователь открывает раздел Планировщика задач.
Приложение:
- формирует параметры задачи;
- создаёт или обновляет задачу в Windows Task Scheduler;
- сохраняет связь профиля с задачей.

### 5. Регламентный запуск
Планировщик Windows запускает CLI-режим приложения.
CLI-режим:
- получает имя профиля или путь к профилю;
- выполняет backup без открытия GUI;
- пишет лог;
- возвращает код завершения.

## Потоки уровня архитектуры

### GUI flow
UI -> Application Service -> Domain Validation -> Infrastructure -> Result -> UI

### Scheduled flow
Task Scheduler -> CLI Entry Point -> Application Service -> Infrastructure -> Log/Exit Code

## Коды завершения

Для CLI-режима должны использоваться явные коды:
- 0 — успех;
- 1+ — ошибка выполнения;
- отдельные коды для ошибок конфигурации, запуска pg_dump, логирования, scheduler.

## Общий принцип

GUI и регламентный запуск должны использовать один и тот же application-слой, чтобы избежать дублирования логики.
