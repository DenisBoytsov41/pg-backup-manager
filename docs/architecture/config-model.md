# Configuration Model

## Назначение документа

Этот документ описывает текущую модель конфигурации проекта PG Backup Manager.

## Уровни конфигурации

Проект поддерживает два основных типа конфигурации:

1. Настройки приложения
2. Профили резервного копирования

---

## 1. Настройки приложения

Настройки приложения относятся к самому GUI и рабочей среде пользователя.

### Текущий состав AppSettings

- `schema_version` — версия схемы настроек приложения;
- `last_profile_path` — путь к последнему открытому профилю;
- `recent_profile_paths` — список недавно открытых профилей;
- `window_width` — ширина окна;
- `window_height` — высота окна.

### Назначение AppSettings

Настройки приложения нужны для:

- восстановления последнего рабочего состояния GUI;
- быстрого доступа к недавно использованным профилям;
- сохранения базовых пользовательских предпочтений интерфейса.

### Формат хранения

Настройки приложения хранятся в отдельном JSON-файле.

---

## 2. Профиль резервного копирования

Профиль backup описывает конкретный сценарий резервного копирования PostgreSQL.

### Общая структура BackupProfile

Профиль включает:

- `schema_version`
- `profile_name`
- `postgres`
- `backup`
- `logging`
- `scheduler`

---

## 2.1 PostgreSQL settings

Секция `postgres` содержит параметры подключения и пути к утилитам PostgreSQL.

### Поля

- `host` — адрес PostgreSQL-сервера;
- `port` — порт PostgreSQL;
- `databases` — список баз данных для резервного копирования;
- `user` — пользователь PostgreSQL;
- `password` — пароль PostgreSQL;
- `pg_dump_path` — путь к `pg_dump.exe`;
- `pg_dumpall_path` — путь к `pg_dumpall.exe`.

### Правила

- `host` должен быть заполнен;
- `port` должен быть числом от 1 до 65535;
- список `databases` должен содержать хотя бы одну базу;
- `user` должен быть заполнен;
- `pg_dump_path` должен быть заполнен;
- `password` может быть пустым;
- `pg_dumpall_path` обязателен, если включена выгрузка globals.

### Примечание по безопасности

Пароль PostgreSQL допускается хранить в профиле, но рекомендуется по возможности не сохранять его без необходимости.

---

## 2.2 Backup settings

Секция `backup` описывает параметры хранения и именования backup-файлов.

### Поля

- `backup_dir` — директория хранения backup-файлов;
- `retention_days` — срок хранения backup-файлов и run-log;
- `dump_globals` — выполнять ли отдельную выгрузку globals;
- `naming_pattern` — шаблон имени dump-файла.

### Правила

- `backup_dir` должен быть заполнен;
- `retention_days` не может быть отрицательным;
- `dump_globals = true` требует указания `pg_dumpall_path`;
- `naming_pattern` должен быть корректным строковым шаблоном.

### Поддерживаемые плейсхолдеры `naming_pattern`

Допустимы только:

- `{database}`
- `{timestamp}`
- `{profile}`

### Обязательные плейсхолдеры

Шаблон имени backup-файла должен содержать:

- `{database}`
- `{timestamp}`

Плейсхолдер `{profile}` необязателен.
