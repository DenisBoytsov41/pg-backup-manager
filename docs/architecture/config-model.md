# Configuration Model

## Назначение документа

Этот документ описывает модель конфигурации проекта PG Backup Manager.

## Уровни конфигурации

Проект должен поддерживать два основных типа конфигурации:

1. Настройки приложения
2. Профили резервного копирования

## 1. Настройки приложения

Настройки приложения относятся к самому GUI и рабочей среде пользователя.

### Примерный состав
- путь к последнему открытому профилю;
- список недавно открытых профилей;
- размер и позиция окна;
- тема интерфейса;
- язык интерфейса;
- предпочтения логирования.

## 2. Профиль резервного копирования

Профиль backup описывает конкретный сценарий резервного копирования.

### Состав профиля

#### PostgreSQL settings
- host
- port
- database или список databases
- user
- password (опционально)
- pg_dump_path
- pg_dumpall_path

#### Backup settings
- backup_dir
- retention_days
- dump_globals
- naming_pattern

#### Logging settings
- main_log_name
- run_log_dir
- log_level

#### Scheduler settings
- enabled
- task_name
- schedule_type
- start_time
- run_user
- run_with_highest_privileges

## Правила проектирования модели

1. Профиль должен быть самодостаточным.
2. Пароль должен быть опциональным.
3. Модель должна поддерживать будущее расширение на несколько баз.
4. Scheduler settings должны быть частью профиля, а не глобальных настроек приложения.

## Формат хранения

На первом этапе конфигурация будет храниться в JSON.
В будущем допускается переход на более сложную схему, если потребуется миграция или versioning профилей.

## Версионирование профилей

Желательно предусмотреть поле версии схемы конфигурации, чтобы в будущем поддерживать миграции формата.
