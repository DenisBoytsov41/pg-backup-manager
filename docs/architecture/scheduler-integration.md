# Scheduler Integration

## Назначение документа

Этот документ описывает, как PG Backup Manager интегрируется с Планировщиком задач Windows (`schtasks`).

## Общий принцип

Регламентный запуск backup выполняется не напрямую из GUI, а через задачу Windows Task Scheduler.

Сценарий работы такой:

1. Пользователь настраивает профиль backup в приложении.
2. Пользователь включает параметры Планировщика в профиле.
3. Приложение создаёт или обновляет задачу Windows Task Scheduler.
4. В нужное время Планировщик запускает PG Backup Manager в CLI-режиме.
5. CLI-режим выполняет команду `run-profile` для указанного JSON-профиля.
6. Приложение выполняет backup и завершает работу с кодом возврата.

## Почему используется именно такой подход

Такой способ интеграции выбран, чтобы:

- использовать стандартный механизм Windows;
- не держать фоновый сервис приложения постоянно запущенным;
- обеспечить единый сценарий выполнения backup как вручную, так и по расписанию;
- использовать один и тот же application-слой для GUI и CLI.

## Источник настроек

Все параметры Планировщика хранятся внутри backup-профиля, в секции `scheduler`.

### Состав scheduler settings

- `enabled` — включён ли Планировщик для профиля;
- `task_name` — имя задачи в Windows Task Scheduler;
- `schedule_type` — тип расписания:
  - `daily`
  - `weekly`
  - `disabled`
- `start_time` — время запуска в формате `HH:MM`;
- `days_of_week` — список дней недели для weekly-режима;
- `run_user` — пользователь Windows, от имени которого запускается задача;
- `run_with_highest_privileges` — запуск с повышенными правами.

## Правила валидации scheduler

Для корректной работы используются следующие правила:

1. Если `enabled = true`, то:
   - `task_name` должен быть заполнен;
   - `schedule_type` не должен быть `disabled`;
   - `start_time` должен быть в формате `HH:MM`;
   - для weekly-расписания должен быть указан хотя бы один день недели.

2. Если `enabled = false`, задача не создаётся и не обновляется.

3. Значение `schedule_type = disabled` допустимо только при выключенном Планировщике.

## Поддерживаемые дни недели

Для weekly-задач приложение поддерживает нормализацию дней недели.

Допустимы английские и русские варианты:

- `MON`, `MONDAY`, `ПН`, `ПОН`, `ПОНЕДЕЛЬНИК`
- `TUE`, `TUESDAY`, `ВТ`, `ВТОРНИК`
- `WED`, `WEDNESDAY`, `СР`, `СРЕДА`
- `THU`, `THURSDAY`, `ЧТ`, `ЧЕТВЕРГ`
- `FRI`, `FRIDAY`, `ПТ`, `ПЯТНИЦА`
- `SAT`, `SATURDAY`, `СБ`, `СУББОТА`
- `SUN`, `SUNDAY`, `ВС`, `ВОСКРЕСЕНЬЕ`

После нормализации значения преобразуются в формат, понятный `schtasks`.

Если приложение запускается из исходников, задача формируется через:

- переход в корень проекта;
- установку `PYTHONPATH=src`;
- запуск команды:
