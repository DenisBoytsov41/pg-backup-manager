# Quick Start

## Назначение

PG Backup Manager — это приложение для настройки, тестового запуска и регламентного выполнения резервного копирования PostgreSQL-баз 1С под Windows.

## Первый запуск

1. Открой приложение.
2. Нажми **Новый**, чтобы создать новый профиль.
3. На вкладке **Конфиг** укажи имя профиля.
4. На вкладке **PostgreSQL** заполни:
   - host;
   - port;
   - список баз через запятую;
   - пользователя PostgreSQL;
   - путь к `pg_dump.exe`;
   - путь к `pg_dumpall.exe`, если нужен экспорт globals.
5. На вкладке **Backup** укажи:
   - папку для хранения backup;
   - срок хранения файлов;
   - имя общего лога.
6. Нажми **Сохранить** или **Сохранить как...** и сохрани профиль в JSON.

## Проверка профиля

На вкладке **Действия** нажми **Проверить профиль**.  
Если настройки заполнены корректно, приложение покажет сообщение об успешной проверке.

## Тестовый backup

На вкладке **Действия** нажми **Тестовый backup сейчас**.

Если backup выполнен успешно, приложение покажет:
- имя профиля;
- папку backup;
- путь к run-log;
- список созданных dump-файлов;
- путь к файлу globals, если он создавался.

## Настройка Планировщика задач

На вкладке **Планировщик** можно:
- включить планировщик для профиля;
- указать имя задачи;
- выбрать тип расписания;
- указать время запуска;
- при недельном расписании указать дни недели через запятую;
- при необходимости указать пользователя запуска.

Для применения настроек:
1. Сохрани профиль.
2. Нажми **Создать / обновить задачу**.

Для проверки задачи:
- нажми **Проверить задачу**.

Для ручного запуска задачи:
- нажми **Запустить задачу сейчас**.

Для удаления задачи:
- нажми **Удалить задачу**.

## Рекомендации

- Лучше не хранить пароль PostgreSQL в профиле без необходимости.
- Для регулярной работы желательно использовать отдельную папку backup.
- Перед включением расписания сначала проверь backup вручную.
