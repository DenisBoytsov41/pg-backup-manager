# Quick Start

## Назначение

PG Backup Manager — это приложение для настройки, тестового запуска и регламентного выполнения резервного копирования PostgreSQL-баз 1С под Windows.

## Что нужно заранее

Перед началом работы желательно подготовить:

- установленный PostgreSQL-клиент или серверный пакет, содержащий `pg_dump.exe`;
- при необходимости выгрузки globals — `pg_dumpall.exe`;
- доступ к PostgreSQL-серверу;
- отдельную папку для хранения backup-файлов.

## Первый запуск

1. Открой приложение.
2. Нажми **Новый**, чтобы создать новый профиль.
3. На вкладке **Конфиг** укажи имя профиля.
4. На вкладке **PostgreSQL** заполни:
   - host;
   - port;
   - список баз через запятую;
   - пользователя PostgreSQL;
   - пароль PostgreSQL, если он нужен;
   - путь к `pg_dump.exe`;
   - путь к `pg_dumpall.exe`, если нужна выгрузка globals.
5. На вкладке **Backup** укажи:
   - папку для хранения backup;
   - срок хранения файлов;
   - шаблон имени dump-файла;
   - имя общего лога.
6. Нажми **Сохранить** или **Сохранить как...** и сохрани профиль в JSON.

## Как заполнять шаблон имени backup-файла

Поле `naming_pattern` определяет имя `.backup`-файла.

### Допустимые плейсхолдеры

Можно использовать:

- `{database}`
- `{timestamp}`
- `{profile}`

### Обязательные плейсхолдеры

Шаблон должен содержать:

- `{database}`
- `{timestamp}`
